// usage

From install to
production in minutes.

from ical_rental_sync import CalendarSync

sync = CalendarSync(
    urls={
        "airbnb":  "https://www.airbnb.com/calendar/ical/YOUR_ID.ics",
        "booking": "https://ical.booking.com/v1/export?t=TOKEN",
    },
    cache_ttl=120,  # cache 2 minutes
)

# Check if a date range is free
if sync.is_available("2026-06-01", "2026-06-07"):
    print("Dates are free!")

# All booked dates as a flat list
booked = sync.get_booked_dates()
# → [date(2026, 6, 1), date(2026, 6, 2), ...]
from ical_rental_sync import CalendarSync
from django.http import JsonResponse

# Initialise once — cache handles the rest
SYNC = CalendarSync(
    urls={"airbnb": "https://...", "booking": "https://..."},
    cache_ttl=120,
)

def check_availability(request):
    check_in  = request.GET.get("check_in")
    check_out = request.GET.get("check_out")
    available = SYNC.is_available(check_in, check_out)
    return JsonResponse({"available": available})

def booked_dates(request):
    booked = SYNC.get_booked_dates()
    return JsonResponse({
        "booked_dates": [d.isoformat() for d in booked]
    })
from celery import shared_task
from myapp.views import SYNC

@shared_task
def refresh_calendars():
    # Pre-warm cache every 5 minutes
    SYNC.clear_cache()
    SYNC.get_combined_availability()

# settings.py
CELERY_BEAT_SCHEDULE = {
    "refresh-rental-calendars": {
        "task": "myapp.tasks.refresh_calendars",
        "schedule": 300,
    }
}
# Option A — only Airbnb
sync = CalendarSync(urls={"airbnb": "https://..."})

# Option B — only Booking.com

// how it works

Three steps.
Done.

01

Install and import

One pip install. One import. No config files needed beyond your iCal URLs.

pip install ical-rental-sync
02

Paste your iCal URLs

Grab the export link from each platform. Airbnb, Booking.com, and VRBO all have them in their calendar settings. Paste them as a dict.

03

Call is_available() and ship

Returns True or False. Plug it into your Django view, Celery task, or any Python backend. Fetching, parsing, merging and caching are all handled.

// api reference

Clean API.
Fully typed.

Every method has type hints and docstrings. Your IDE knows everything before you do.

MethodReturnsDescription
is_available(check_in, check_out)boolTrue if no platform has a booking overlapping the range
get_combined_availability()list[BookedPeriod]All blocked periods from all platforms, merged and sorted
get_booked_dates()list[date]Flat list of every individual booked date — great for calendar widgets
clear_cache()NoneInvalidate all cached data — next call fetches fresh
add_url(label, url)NoneAdd a new iCal feed at runtime without reinstantiation
remove_url(label)NoneRemove a feed by its label
